## EsperantAIとは？

EsperantAIは、人工知能を使って顔と手のジェスチャーをリアルタイムに検出し、配信ソフトのコマンドに変換する**ウェブアプリ**です。以下のソフトウェアに対応しています：

- **OBS Studio** 28+
- **Streamlabs Desktop**
- **vMix**
- **PRISM Live Studio**
- **XSplit Broadcaster**（ベータ）

また、以下のプラットフォームからイベントを受信できます：

- **Twitch**
- **YouTube Live**
- **Kick**
- **Trovo**
- **StreamElements**（マルチプラットフォームブリッジ）

### なぜ「誠実なジェスチャー」なのか？

基本的な顔の表情や頭の回転は、**すべての人間文化において共通**しています（Paul Ekman, 1972）。嘘をつかず、地域によって変わりません。EsperantAIではこれらを「🌐 ユニバーサル」ジェスチャーと呼び、意味が国によって異なる可能性のある「⚠️ カルチャー」ジェスチャー（手話サイン）と区別しています。

ご自身の視聴者に合わせて、どのジェスチャーを使うかお選びください。

## 初めての使用

### ステップ1：カメラアクセスを許可

初めてEsperantAIを開くと、ブラウザがカメラの使用許可を求めます。**許可してください**。

> 重要：EsperantAIは映像を一切サーバーに送信しません。処理はすべてお使いのコンピューター上で100%ローカルに行われます。

### ステップ2：カメラを選択

カメラが複数ある場合は、ドロップダウンから使用するカメラを選択してください。

### ステップ3：検出の確認

左パネルに顔が表示されます。EsperantAIが顔を検出すると、Yaw / Pitch / Roll のインジケーターに数値が表示されます。

### ステップ4：キャリブレーションウィザード（Pro+）

ProまたはPro+ライセンスをお持ちの場合、初回使用時に**キャリブレーションウィザード**が自動的に起動します。自然な可動範囲を測定し、最適な感度を設定します。**再キャリブレーション**ボタンからいつでも再実行できます。

## 配信ソフトの接続

### OBS Studio

1. OBSで：**ツール → WebSocket Server Settings**
2. WebSocketを有効にする。パスワードを設定した場合はメモしておく
3. EsperantAIで：**接続**パネル
4. 配信ソフト：**OBS Studio**
5. WebSocket URL：`ws://127.0.0.1:4455`（デフォルト）
6. パスワード：OBSで設定したもの
7. **Connect** をクリック

### Streamlabs Desktop

1. Streamlabsで：**Settings → Remote Control**
2. Remote Controlを有効にする
3. API Tokenをメモ
4. EsperantAIで：配信ソフト：**Streamlabs Desktop**
5. API Token：貼り付け
6. ポート：`59650`（デフォルト）
7. **Connect** をクリック

### vMix

1. vMixで：**Settings → Web Controller**
2. Web Controllerを有効にする。デフォルトポート：8088
3. EsperantAIで：配信ソフト：**vMix**
4. ホスト：`127.0.0.1`
5. ポート：`8088`
6. **Connect** をクリック

### PRISM Live Studio

1. PRISM Live Studio v4.0.5以降ではobs-websocketプラグインの手動インストールが必要
2. [OBSフォーラム](https://obsproject.com/forum/resources/obs-websocket-remote-control-of-obs-studio-made-easy.466/)から`obs-websocket`をダウンロード
3. PRISMのプラグインフォルダーにコピー
4. PRISMを再起動
5. **ツール → WebSocket Server Settings** でWebSocketを有効にする
6. EsperantAIで：配信ソフト：**PRISM Live Studio**（OBSと同じ方法で動作）

### XSplit Broadcaster（ベータ）

1. XSplitで「Remote xjs」拡張機能をインストール（Settings → Extensions）
2. 設定でRemoteを有効にする
3. EsperantAIで：配信ソフト：**XSplit**
4. Remote xjs Proxy URL：`ws://127.0.0.1:5555/xjs`（デフォルト）
5. **Connect** をクリック

> XSplitは**ベータ版**です。高度な機能が制限される場合があります。

## ジェスチャーとシーンの設定

接続すると、配信ソフトの実際のシーンが**トリガー**パネルのドロップダウンに自動的に表示されます。

### 基本的なマッピング

1. 各ジェスチャー（例：「左を向く」）に対して、ドロップダウンからシーンを選択
2. そのジェスチャーを行い、約150ms安定して保つと、EsperantAIが配信ソフトのシーンを切り替えます
3. 切り替えは自動的かつほぼ瞬時に行われます

### マルチアクション（Pro+）

ProまたはPro+ライセンスでは、1つのジェスチャーで**複数のアクション**を同時にトリガーできます：
- シーン切り替え＋音声再生＋オーバーレイ表示＋チャットメッセージ送信

### カテゴリーの有効化/無効化

各カテゴリーには個別の「有効化」チェックボックスがあります：

- 🧠 **頭の回転**（ユニバーサル — デフォルトで有効）
- 📏 **顔の距離**（近づく/遠ざかる）
- 👁️ **視線**（目だけを動かす）
- 😀 **感情**（笑顔、驚き、怒り、ニュートラル）
- 👁️‍🗨️ **ダブルブリンク**
- ✋ **手のジェスチャー**（カルチャー — デフォルトで無効）

CPUを節約するため、不要なカテゴリーは無効にしてください。

| ニュートラル | 感情=neutral | リラックスした表情 |

### ⚠️ カルチャー（国によって意味が異なる）

| ジェスチャー | 西洋での意味 | 他文化での注意点 |
|---|---|---|
| 👍 サムズアップ | 賛成・OK | 中東/西アジア：攻撃的に受け取られる場合あり |
| ✌️ ピース | 平和・勝利 | 英国/アイルランド/オーストラリア（手のひらを内側に）：侮辱 |
| 🤘 ロックサイン | ロック・メタル | イタリア（手のひらを下に）：「コルヌート」（侮辱）。※日本では特に問題なし |
| 👌 OK | OK・完璧 | ブラジル/トルコ/ドイツ：攻撃的に受け取られる場合あり。※日本では問題なく使用可能 |
| ✊ 握りこぶし | 文脈により異なる | — |
| 🖐️ 開いた手のひら | 「ストップ」または挨拶 | ギリシャ（相手に向けてmountza）：強い侮辱 |
| ☝️ 指差し | 指示 | アジア：指で指すのは失礼 |

### 🙏 合掌（合掌）

特別なジェスチャー：胸の前で両手のひらを合わせます（祈りや挨拶のお辞儀のように）。合掌（がっしょう）は日本の文化において、敬意や感謝の意を表す一般的なジェスチャーであり、仏教の礼拝や日常の挨拶（お辞儀と共に）でも広く使われています。6つのランドマークチェックにより高い信頼性で検出されます。

## 配信プラットフォームの接続

EsperantAIがイベント（投げ銭、サブスク、レイド）を受信するには、配信しているプラットフォームを接続してください。

### Twitch

1. https://dev.twitch.tv/console でClient IDを作成
2. リダイレクトURIを登録：`https://esperantai.com/oauth-callback.html`（またはローカルURL）
3. EsperantAIで：**プラットフォームイベント**パネル → **Twitch EventSub**
4. Client IDを貼り付け
5. **Connect** をクリック
6. Twitchの認証ウィンドウが開きます。権限を許可
7. ウィンドウが閉じ、「Twitch Connected」と表示されます

### YouTube Live

1. https://console.cloud.google.com で認証情報を作成
2. YouTube Data API v3を有効にする
3. OAuth Client IDを作成（種類：ウェブアプリケーション）
4. Twitchと同じリダイレクトURIを登録
5. EsperantAIで：**プラットフォームイベント**パネル → **YouTube Live**
6. Client IDを貼り付け、**Connect** をクリック

### Kick

1. https://kick.com/settings/developer でアプリを作成
2. リダイレクトURIを登録
3. EsperantAIで：**プラットフォームイベント**パネル → **Kick**
4. Client IDを貼り付け、**Connect** をクリック
5. KickはOAuth 2.1 with PKCEを使用（より安全）

### StreamElements（マルチプラットフォームブリッジ）

すでにStreamElementsのアカウントをお持ちの場合、1つのトークンでTwitch + YouTube + Facebookを統合できます：

1. https://streamelements.com/dashboard/account/channels にアクセス
2. JWT Tokenをコピー
3. EsperantAIで：**プラットフォームイベント**パネル → **StreamElements**
4. JWTを貼り付け、**Connect** をクリック

## イベント＋ジェスチャーの組み合わせ（上級）

EsperantAIの魔法はここにあります：**プラットフォームイベント**と**あなたのジェスチャー**を確認として組み合わせます。

### 例：投げ銭にサムズアップで感謝

1. **イベントトリガー**パネル → 「💰 投げ銭」の行
2. ✅ 有効化
3. シーン：`Thank_You_Scene`
4. 必須ジェスチャー：`👍 サムズアップ`

**ライブの流れ**：
- 投げ銭が届く → EsperantAIに「ジェスチャー待ち...」と表示
- 5秒以内に👍を行う
- 行うと → `Thank_You_Scene`に切り替え＋設定された他のアクションを実行
- 行わないと → 自動的に却下

### 必須ジェスチャーなし（自動トリガー）

「必須ジェスチャー」を`— なし —`のままにすると、イベントが即座にアクションをトリガーします。

以下の場面で便利：
- レイド到着時に自動でお祝いシーンに切り替え
- サブスク時に自動でオーバーレイを表示

## 感度とデッドゾーン

### 感度

しきい値はジェスチャーがトリガーされるのにどれくらいの大きさが必要かを制御します：

- **Yaw**：頭を横にどれくらい回すか（デフォルト：0.15 rad ≈ 8.6°）
- **Pitch 上/下**：上下の傾き
- **Roll**：左右の傾き

値を上げるとより大きなジェスチャーが必要に、下げるとより感度が高くなります。

### デッドゾーン（疲労防止）

ほぼ中央を向いている場合（yaw < 0.05、pitch < 0.05、roll < 0.08）、**何もトリガーされません**。これにより、自然な動きでマイクロモーションがトリガーされるのを防ぎます。

### 安定フレーム

`安定フレーム` = ジェスチャーがトリガーされる前に連続して何フレーム保持しなければならないか。デフォルト：5フレーム（約150ms / 30fps）。

トリガーが簡単に発動しすぎる場合は値を上げ、より速い反応が必要な場合は下げてください。

### クールダウン

`クールダウン (ms)` = シーン切り替え間の最短時間。デフォルト：500ms。

素早く揺れ動いた場合の切り替えの「ジッター」を防ぎます。

## トラブルシューティング

### ライセンスキーを貼り付けたのに「認証が必要」と表示される

- キーを完全にコピーしたか確認（4文字×5グループ、ハイフン区切り）
- インターネット接続を確認（初回認証にはオンライン検証が必要）
- すでに3台で認証済みの場合は、先に1台を無効化
- 解決しない場合は soporte@edugame.digital に連絡

### 顔が見えているのに「顔を検索中...」のまま

- 照明を改善：顔が明るく照らされているか確認
- カメラに近づく（40〜80cmが最適）
- GPUを使用する他のタブを閉じる（Chromeはタブが多すぎるとGPUを制限することがあります）
- Chromeのメモリセーバーが有効な場合、このタブで無効にする

### ドロップダウンにシーンが表示されない

- 配信ソフトに接続されているか確認（緑のバッジ「Connected」）
- `R`キーを押してシーンリストを再読み込み
- それでも空の場合は、切断して再接続

### ジェスチャーをしていないのにシーンが切り替わる

- **感度**パネルでyaw / pitch / rollのしきい値を上げる
- `安定フレーム`を5から8〜10に上げる
- デッドゾーンが設定されているか確認（yaw 0.05、pitch 0.05、roll 0.08）
- 他人がフレーム内にいないか確認（複数顔で不安定になることがあります）

### 検出のラグ

- 重いアプリ（ゲーム、動画編集）を閉じる
- 統合GPUではなく、専用GPUを使用しているか確認
- カメラ解像度が4Kの場合は1080pに下げる（検出には1080pが最適）

### EsperantAIに「シーン変更」と表示されるのにOBSが反応しない

- ドロップダウンのシーン名がOBSのものと完全に一致しているか確認（大文字小文字を区別）
- シーンが別のシーンコレクションにないか確認
- **トリガー履歴**パネルを確認 — ✗赤が表示されている場合、具体的なエラーがあります

### 「OBSに接続できません — 手動で接続してください」というエラー

- OBSが開いているか確認
- OBSでWebSocketが有効になっているか確認
- OBSでパスワードを設定している場合、正確に一致している必要があります
- 一部のウイルス対策ソフトがポート4455をブロック — 例外を追加

## プライバシー

### EsperantAIがしないこと

- ❌ 映像をサーバーに送信しません
- ❌ 映像やキャプチャを保存しません
- ❌ 生体情報をリモートで収集しません
- ❌ データを広告主や第三者と共有しません

### 処理する内容

- ✅ ブラウザ内でのローカル顔検出（Human.js + WebGL）
