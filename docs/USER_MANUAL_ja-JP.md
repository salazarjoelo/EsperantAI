## EsperantAIとは？

EsperantAIは、AIで顔と手のジェスチャーをリアルタイムに検出し、配信ソフトのコマンドに変換する**ウェブアプリ**です。カメラ映像はブラウザ内でローカル処理されます。

![EsperantAIのローカル処理フロー](assets/manual/01-esperantai-flow.svg)

以下の配信ソフトで使えます。

- **OBS Studio** 28+
- **Streamlabs Desktop**
- **vMix**
- **PRISM Live Studio**
- **XSplit Broadcaster**（ベータ/上級者向け）

プラットフォームのイベントを受け取り、ジェスチャーと組み合わせることもできます。

- **Twitch**: EventSub WebSocketによる直接対応。
- **YouTube Live**: YouTube Data API v3による直接対応。アクティブなライブ配信と利用可能なクォータが必要です。
- **Kick**: ローカルの **Streamer.bot ブリッジ** 経由で対応します。EsperantAI はブラウザに Kick のシークレットを保存しません。
- **StreamElements**: アカウントのトークン/JWTを使うマルチプラットフォームブリッジです。
- **Trovo**: Trovo OAuth と公式チャット WebSocket による直接サポート。

### なぜ「誠実なジェスチャー」なのか？

基本的な表情や頭の回転は、**人間の文化全体に共通する**ものです（Paul Ekman, 1972）。地理によって意味が変わりにくく、偽りにくい身体信号です。EsperantAIではこれらを「🌐 ユニバーサル」ジェスチャーと呼び、国によって意味が変わることのある「⚠️ カルチャー」ジェスチャー（手のサイン）と区別しています。

どのジェスチャーを使うかは、あなたの視聴者に合わせて選べます。

## 最小要件

### ハードウェア

- **任意のUSBウェブカメラ**（推奨: 1080p以上）
- **CPU**: 直近5年程度の4コア以上のプロセッサー
- **RAM**: 最低8 GB。同時に配信する場合は16 GB推奨。
- **GPU**: WebGL対応GPU（最近の内蔵GPUでも動作します）

### ソフトウェア

- **OS**: Windows 10/11、macOS 12+、または新しいカーネルのLinux
- **ブラウザ**: Chrome 90+、Edge 90+、またはFirefox 100+
- **配信ソフト**（少なくとも1つ）: OBS Studio 28+、Streamlabs Desktop、vMix、PRISM、XSplit

### インターネット

- **ライセンス認証**と**7日ごとの再検証**に必要です
- **最大7日間はオフラインでも動作**します（猶予期間）

2. 新しいデバイス: 通常どおり認証

## 初めて使うとき

### ステップ1: カメラアクセスを許可する

EsperantAIを初めて開くと、ブラウザがカメラへのアクセス許可を求めます。**許可**してください。

> 重要: EsperantAIはあなたの映像をサーバーへ送信しません。処理は100%お使いのデバイス内で行われます。

### ステップ2: カメラを選択する

複数のカメラがある場合は、カメラのドロップダウンから使うカメラを選びます。

### ステップ3: 検出を確認する

左側のパネルに顔が表示されます。EsperantAIが顔を検出すると、Yaw / Pitch / Roll のインジケーターに値が表示され始めます。

### ステップ4: キャリブレーションウィザード（Pro+）

ProまたはPro+ライセンスをお持ちの場合、初回使用時に**キャリブレーションウィザード**が自動で起動します。自然に動かせる範囲を測定し、最適な感度を設定します。**再キャリブレーション**ボタンからいつでも再実行できます。

## 配信ソフトを接続する

![配信ソフト接続マトリクス](assets/manual/02-software-setup.svg)

このセクションの接続はすべてローカル接続です。EsperantAIは、同じコンピューター上で動作している配信ソフトと `127.0.0.1` 経由で通信します。

### OBS Studio

1. OBSで **ツール → WebSocketサーバー設定** を開きます
2. WebSocketサーバーを有効にします。OBS Studio 28+にはobs-websocketが含まれています
3. EsperantAIで **接続** パネルを開きます
4. 配信ソフト: **OBS Studio**
5. WebSocket URL: `ws://127.0.0.1:4455`（既定値）
6. パスワード: OBSでパスワードを有効にした場合は、そのパスワード
7. **接続** をクリックします

### Streamlabs Desktop

1. Streamlabs Desktopで **Settings → Remote Control** を開きます
2. ローカルのRemote Controlを有効にします
3. Remote Control画面から **API Token** をコピーします
4. EsperantAIで配信ソフトに **Streamlabs Desktop** を選びます
5. API Tokenを貼り付けます
6. ポート: `59650`（既定値）
7. **接続** をクリックします

### vMix

1. vMixで **Settings → Web Controller** を開きます
2. Web Controllerを有効にします。既定のポートは `8088` です
3. EsperantAIで配信ソフトに **vMix** を選びます
4. ホスト: `127.0.0.1`
5. ポート: `8088`
6. **接続** をクリックします

> 注: 現在のEsperantAIアダプターは、vMixのローカルHTTP APIを使用します。Web Controllerをネットワークルールやブラウザ非対応の認証情報で保護している場合、接続に失敗することがあります。

### PRISM Live Studio

1. **PRISM Live Studio v4.0.5+** を使用します
2. OBS/PRISM互換の `obs-websocket` プラグインを手動でインストールします
3. PRISMの公式OBSプラグインガイドに従って、PRISMのプラグインフォルダーへコピーします
4. PRISMを再起動します
5. **ツール → WebSocketサーバー設定** でWebSocketを有効にします
6. EsperantAIで配信ソフトに **PRISM Live Studio** を選びます（OBSと同じ方式で動作します）

> 重要な違い: OBS 28+にはobs-websocketが含まれています。PRISMではプラグインを手動でインストールする必要があります。

### XSplit Broadcaster（ベータ/上級者向け）

1. **XSplit XJS / Remote xjs** と互換性のあるローカルブリッジをインストールまたは有効化します
2. ブリッジがローカルWebSocket URLを公開していることを確認します
3. EsperantAIで配信ソフトに **XSplit** を選びます
4. Remote xjsプロキシURL: `ws://127.0.0.1:5555/xjs`（既定値）
5. **接続** をクリックします

> XSplitは**ベータ/上級者向け**です。互換性はインストール済みのローカルXJSブリッジに依存し、高度な機能は制限される場合があります。

## ジェスチャーとシーンを設定する

接続が完了すると、配信ソフト内の実際のシーンが **トリガー** パネルのドロップダウンに自動表示されます。

### 基本マッピング

1. 各ジェスチャー（例: 「左を見る」）に対して、ドロップダウンからシーンを選びます
2. そのジェスチャーを行い、約150ms安定して保つと、EsperantAIが配信ソフトのシーンを切り替えます
3. 切り替えは自動で、ほぼ即時に行われます

### マルチアクション（Pro+）

ProまたはPro+ライセンスでは、1つのジェスチャーから**複数のアクション**を同時に実行できます。
- シーン切り替え + サウンド再生 + オーバーレイ表示 + チャットメッセージ送信

### カテゴリーの有効化 / 無効化

各カテゴリーには専用の「有効化」チェックボックスがあります。

- 🧠 **頭の回転**（ユニバーサル — 既定で有効）
- 📏 **顔の距離**（近づく / 離れる）
- 👁️ **視線**（目だけを動かす）
- 😀 **感情**（笑顔、驚き、怒り、ニュートラル）
- 👁️‍🗨️ **ダブルまばたき**
- ✋ **手のジェスチャー**（カルチャー — 既定で無効）

CPU使用量を抑えたい場合は、不要なカテゴリーを無効にしてください。

## ジェスチャーカテゴリー

### 🌐 ユニバーサル（どの文化でも同じ意味）

| ジェスチャー | 軸 | 有効にする動き |
|---|---|---|
| センター | — | 正面を向き、顔を安定させる |
| 左を見る | yaw 負方向 | 頭を左に向ける |
| 右を見る | yaw 正方向 | 頭を右に向ける |
| 上を見る | pitch 負方向 | 顔を上げる |
| 下を見る | pitch 正方向 | 顔を下げる |
| 左に傾ける | roll 負方向 | 頭を左肩の方へ傾ける |
| 右に傾ける | roll 正方向 | 頭を右肩の方へ傾ける |
| 近づく | 距離 | カメラに近づく |
| 離れる | 距離 | カメラから離れる |
| 視線 | 視線 | 頭を中央に保ったまま目だけを動かす |
| 笑顔 | 感情=happy | はっきり笑う |
| 驚き | 感情=surprise | 驚いた表情を見せる |
| 怒り | 感情=anger | 怒った表情を見せる |
| ニュートラル | 感情=neutral | 力を抜いた表情 |
| ダブルまばたき | まばたき | 両目をすばやく2回閉じる（< 700ms） |

### ⚠️ カルチャー（意味が国によって変わる）

| ジェスチャー | 西洋での意味 | 他文化での注意点 |
|---|---|---|
| 👍 親指を立てる | 承認 | 中東 / 西アジアでは侮辱的に受け取られる場合があります |
| ✌️ ピース | 平和 / 勝利 | 英国 / アイルランド / オーストラリアで手のひらが内側の場合は侮辱になります |
| 🤘 ロックサイン | ロック / メタル | イタリアで手のひらを下に向けると「cornuto」（侮辱）になる場合があります |
| 👌 OK | OK / 完璧 | ブラジル / トルコ / ドイツでは侮辱的に受け取られる場合があります |
| ✊ 握りこぶし | 政治的文脈により変わります | — |
| 🖐️ 開いた手のひら | 「止まれ」または挨拶 | ギリシャで相手に向けると mountza という強い侮辱になります |
| ☝️ 指差し | 示す | アジアでは指で指すことが失礼とされる場合があります |

EsperantAIは、インターフェース上で各ジェスチャーに対応するバッジを表示します。世界中の視聴者に合わせて、使うジェスチャーを選んでください。

### 🙏 Gassho（合掌）

特別なジェスチャーです。祈りやお辞儀の挨拶のように、胸の前で両手のひらを合わせます。東アジアの文化では、敬意や感謝を表すしぐさとして広く使われます。6つのランドマーク確認により、高い信頼性で検出されます。

## 配信プラットフォームを接続する

EsperantAIでイベント（投げ銭、サブスク、レイド、フォロー、Super Chatなど）を受け取るには、配信しているプラットフォームを接続します。

![プラットフォームイベントの状態](assets/manual/03-platform-events.svg)

### Twitch

1. https://dev.twitch.tv/console でClient IDを作成します
2. リダイレクトURIを登録します: `https://TU-DOMINIO/oauth-callback.html`（またはローカルURL）
3. EsperantAIで **プラットフォームイベント** パネル → **Twitch EventSub** を開きます
4. Client IDを貼り付けます
5. **接続** をクリックします
6. Twitchの認証ウィンドウが開きます。権限を許可します
7. ウィンドウが閉じ、「Twitch接続済み」と表示されます

EsperantAIはEventSub WebSocketを使用します。Client Secretはブラウザに貼り付けないでください。

### YouTube Live

1. https://console.cloud.google.com で認証情報を作成します
2. YouTube Data API v3を有効にします
3. OAuth Client IDを作成します（種類: ウェブアプリケーション）
4. Twitchと同じリダイレクトURIを登録します
5. EsperantAIで **プラットフォームイベント** パネル → **YouTube Live** を開きます
6. Client IDを貼り付けて **接続** をクリックします

YouTubeの要件: チャットが利用できるアクティブなライブ配信が必要です。また、Google Cloudプロジェクトにライブチャットを取得するための十分なクォータが必要です。

### Streamer.bot 経由の Kick

EsperantAI は **Streamer.bot ブリッジ** を通じて Kick イベントを受け取ります。ブラウザに Kick のシークレットを露出せず、リバースエンジニアリングに依存しないため、販売用として推奨される方式です。

1. Streamer.bot 1.0.0 以降をインストールします。
2. Streamer.bot で Kick アカウントを接続します。
3. Streamer.bot の **Servers/Clients -> WebSocket Server** を有効化します。
4. 変更していなければ、`127.0.0.1`、ポート `8080`、endpoint `/` を使います。
5. EsperantAI の **プラットフォームイベント** パネル -> **Kick via Streamer.bot** を開きます。
6. **接続** をクリックします。

利用できるイベントは Streamer.bot 側で有効な Kick 連携に依存します。Kick 公式の backend/webhook 連携は高度なロードマップ項目です。

### StreamElements（マルチプラットフォームブリッジ）

すでにStreamElementsアカウントを持っている場合、複数プラットフォームのアラート用ブリッジとして使用できます。

1. https://streamelements.com/dashboard/account/channels にアクセスします
2. JWT Tokenをコピーします
3. EsperantAIで **プラットフォームイベント** パネル → **StreamElements** を開きます
4. JWTを貼り付けて **接続** をクリックします

このトークンは非公開にしてください。StreamElementsアカウントのパスワードと同じように扱ってください。

### Trovo

EsperantAI は OAuth と Trovo 公式チャット WebSocket により Trovo と連携します。

1. Trovo 開発者ポータルでアプリを作成します。
2. EsperantAI を開く同じドメインでリダイレクト URI `https://TU-DOMINIO/oauth-callback.html` を登録します。
3. EsperantAI の **プラットフォームイベント** パネル -> **Trovo** を開きます。
4. Client ID を貼り付け、**接続** をクリックします。
5. 要求された権限を許可します。

利用できるイベントは Trovo のチャットメッセージと公式チャットトークンフローに依存します。

## イベント + ジェスチャーの組み合わせ（上級）

ここがEsperantAIの特徴です。**プラットフォームイベント**と**あなたのジェスチャー**を確認操作として組み合わせます。

![イベントとジェスチャーのフロー](assets/manual/04-event-gesture-combo.svg)

### 例: 投げ銭に親指を立てて感謝する

1. **イベントトリガー** パネル → 「💰 投げ銭」の行
2. ✅ 有効化
3. シーン: `Escena_Gracias`
4. 必須ジェスチャー: `👍 親指を立てる`

**ライブ中の流れ**:
- 投げ銭が届く → EsperantAIが「ジェスチャー待ち...」を表示
- 5秒以内に 👍 を行う
- 実行した場合 → `Escena_Gracias` に切り替え + 設定済みの他のアクションも実行
- 実行しない場合 → 自動的に破棄

### 必須ジェスチャーなし（自動トリガー）

「必須ジェスチャー」を `— なし —` のままにすると、イベントがただちにアクションを実行します。

便利な例:
- レイドが来たときに、お祝いシーンへ自動で切り替える
- サブスク時にオーバーレイを自動表示する

## 感度とデッドゾーン

### 感度

しきい値は、どれくらい大きなジェスチャーでトリガーするかを制御します。

- **Yaw**: 頭を左右にどれくらい回す必要があるか（既定値: 0.15 rad ≈ 8.6°）
- **Pitch 上/下**: 縦方向の傾き
- **Roll**: 横方向の傾き

値を上げると、より大きなジェスチャーが必要になります。値を下げると、より敏感に反応します。

### デッドゾーン（疲労防止）

ほぼ中央にいる場合（yaw < 0.05、pitch < 0.05、roll < 0.08）、**何もトリガーされません**。自然な動きで小さな揺れがトリガーを発動しないようにするためです。

### 安定フレーム

`安定フレーム` = トリガー前に、ジェスチャーを何フレーム連続で維持する必要があるか。既定値: 5フレーム（30fpsで約150ms）。

トリガーが簡単に発動しすぎる場合は値を上げます。反応を速くしたい場合は値を下げます。

### クールダウン

`クールダウン (ms)` = シーン切り替え間の最短時間。既定値: 500ms。

すばやく揺れたときに切り替えが不安定になるのを防ぎます。

## トリガー履歴

**高度な設定 → トリガー履歴** パネルには、直近50件の実行アクションが表示されます。

- ✓ 緑 = 成功
- ✗ 赤 = 失敗
- · グレー = 保留中

DevToolsを開かずに、何がトリガーされたかを確認するのに便利です。

**CSVエクスポート**: 履歴をダウンロードしてオフライン分析できます。

**クリア**: 履歴を削除します（ほかの設定には影響しません）。

## トラブルシューティング

### ライセンスキーを貼り付けた後も「認証が必要」と表示される

- キー全体をコピーしたか確認します（ハイフンで区切られた4文字×5グループ）
- インターネット接続を確認します（初回認証にはオンライン検証が必要です）
- すでに3台で認証している場合は、先に1台を無効化します
- 問題が続く場合は soporte@edugame.digital へ連絡してください

### 顔が見えているのに「顔を検索中...」が続く

- 照明を改善します。顔が十分に明るく見える必要があります
- カメラに近づきます（40〜80 cmが最適です）
- GPUを使う他のタブを閉じます（タブが多すぎるとChromeがGPUを制限することがあります）
- Chromeのメモリセーバーが有効な場合は、このタブでは無効にします

### ドロップダウンにシーンが表示されない

- 配信ソフトに接続していることを確認します（緑の「接続済み」バッジ）
- `R` を押してシーン一覧を再読み込みします
- それでも空の場合は、一度切断してから再接続します
- vMixでは、Web Controllerが有効で `http://127.0.0.1:8088/api/` からアクセスできることを確認します
- PRISMでは、obs-websocketプラグインがインストールされ有効になっていることを確認します
- XSplitでは、ローカルXJSブリッジが実行中であることを確認します

### ジェスチャーをしていないのにシーンが切り替わる

- **感度** パネルで yaw / pitch / roll のしきい値を上げます
- `安定フレーム` を5から8〜10へ上げます
- デッドゾーンが設定されていることを確認します（yaw 0.05、pitch 0.05、roll 0.08）
- 画面内にほかの人がいないことを確認します（複数の顔で不安定になることがあります）

### 検出が遅い

- 重いアプリを閉じます（ゲーム、動画編集など）
- 専用GPUがある場合は、内蔵GPUではなく専用GPUを使っていることを確認します
- カメラ解像度が4Kの場合は下げます（検出には1080pが最適です）

### EsperantAIでは「シーン変更」と表示されるのにOBSが反応しない

- ドロップダウン内のシーン名がOBSのシーン名と**完全に一致**しているか確認します（大文字小文字を区別します）
- そのシーンが別のシーンコレクションにないか確認します
- **トリガー履歴** パネルを確認します。赤い ✗ が表示される場合、具体的なエラーがあります

### 「OBSに到達できません — 手動で接続してください」というエラー

- OBSが開いていることを確認します
- OBSでWebSocketが有効になっていることを確認します
- OBSでパスワードを設定した場合は、完全に一致している必要があります
- 一部のウイルス対策ソフトがポート4455をブロックすることがあります。例外を追加してください

### TwitchまたはYouTubeが接続できない

- プラットフォームのコンソールに登録したリダイレクトURIが、`oauth-callback.html` のURLと完全に一致していることを確認します
- EsperantAIを使っているドメインでポップアップを許可します
- TwitchではClient IDだけを使用し、Client Secretは貼り付けないでください
- YouTubeでは、YouTube Data API v3が有効で、アクティブなライブ配信があることを確認します

### Kick が Streamer.bot 経由で接続できない

Streamer.bot 1.0.0+ が起動していること、Streamer.bot 内で Kick が接続済みであること、**WebSocket Server** が有効であることを確認してください。設定を変更していなければ `127.0.0.1:8080/` を使います。Streamer.bot がパスワードを要求する場合は、同じパスワードを EsperantAI に入力してください。

## プライバシー

### EsperantAIがしないこと

- ❌ あなたの映像をサーバーへ送信しません
- ❌ 映像やキャプチャを保存しません
- ❌ 生体情報をリモートで収集しません
- ❌ データを広告主や第三者と共有しません

### EsperantAIが処理すること

- ✅ ブラウザ内でのローカル顔検出（Human.js + WebGL）
- ✅ OBS / Streamlabs / vMix / PRISM / XSplitへのローカル接続（ループバック `127.0.0.1`）
- ✅ ライセンスキーの定期検証（7日ごと）
- ✅ Twitch/YouTube/Kick/StreamElementsを接続した場合: ブラウザのローカルストレージまたはセッションストレージ内のプラットフォームトークン

詳細は `docs/PRIVACY.html` を参照してください。
