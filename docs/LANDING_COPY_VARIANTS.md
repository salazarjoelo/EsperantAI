## Recomendación de A/B test

- **Métrica primaria:** porcentaje de visitantes que hacen clic en CTA primario.
- **Métrica secundaria:** tiempo en página y porcentaje que llega al bloque de precio.
- **Muestra mínima:** 500 visitantes por variante.
- **Tool sugerido:** PostHog gratis o una solución simple propia con parámetros UTM. Google Optimize fue retirado; no dependería de él para una implementación nueva.
- **Segmentación inicial:** separar tráfico orgánico, tráfico de comunidad streamer y tráfico pagado si se usan ads.

## Hipótesis

- **A es baseline conservador:** mantiene la narrativa actual y mejora claridad sin mover demasiado el posicionamiento.
- **B es la variante más fuerte para conversión temprana:** explica el beneficio antes que el concepto filosófico de “gestos honestos”.
- **C puede funcionar para campañas:** útil si Joel confirma precio fundador, cupos o casos reales, pero no debe publicar escasez ni social proof inventado.

## Recomendación final

Probaría primero **Variant B** contra **Variant A**. B comunica el resultado para el streamer con menos abstracción y debería convertir mejor en audiencias que todavía no conocen EsperantAI. Guardaría C para lanzamiento público cuando existan datos reales de usuarios, screenshots o una oferta fundada en hechos.
