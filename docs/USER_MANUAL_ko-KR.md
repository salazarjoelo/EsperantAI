## EsperantAI란?

EsperantAI는 인공지능을 사용하여 얼굴과 손의 제스처를 실시간으로 감지하고, 스트리밍 소프트웨어의 명령으로 변환하는 **웹 앱**입니다. 다음 소프트웨어와 함께 작동합니다:

- **OBS Studio** 28+
- **Streamlabs Desktop**
- **vMix**
- **PRISM Live Studio**
- **XSplit Broadcaster** (베타)

그리고 다음 플랫폼에서 이벤트를 수신합니다:

- **Twitch**
- **YouTube Live**
- **Kick**
- **Trovo**
- **StreamElements** (멀티플랫폼 브릿지)

### 왜 "진실한 제스처"인가요?

기본적인 얼굴 표정과 머리 회전은 **모든 인간 문화에서 보편적**입니다 (Paul Ekman, 1972). 거짓이 없고 지역에 따라 달라지지 않습니다. EsperantAI는 이를 "🌐 유니버설" 제스처라고 부르며, 의미가 국가에 따라 다를 수 있는 "⚠️ 컬처럴" 제스처(손동작)와 구분합니다.

시청자층에 맞춰 어떤 제스처를 사용할지 직접 선택하세요.

## 처음 사용하기

### 1단계: 카메라 접근 허용

처음 EsperantAI를 열면 브라우저가 카메라 사용 권한을 요청합니다. **허용**해 주세요.

> 중요: EsperantAI는 영상을 어떤 서버에도 전송하지 않습니다. 모든 처리는 컴퓨터에서 100% 로컬로 이루어집니다.

### 2단계: 카메라 선택

카메라가 여러 대인 경우, 드롭다운에서 사용할 카메라를 선택하세요.

### 3단계: 감지 확인

왼쪽 패널에 얼굴이 표시됩니다. EsperantAI가 얼굴을 감지하면 Yaw / Pitch / Roll 표시기에 수치가 나타납니다.

### 4단계: 캘리브레이션 마법사 (Pro+)

Pro 또는 Pro+ 라이선스가 있는 경우, 처음 사용 시 **캘리브레이션 마법사**가 자동으로 실행됩니다. 자연스러운 가동 범위를 측정하여 최적의 감도를 설정합니다. **재캘리브레이션** 버튼으로 언제든 다시 실행할 수 있습니다.

## 스트리밍 소프트웨어 연결

### OBS Studio

1. OBS에서: **도구 → WebSocket 서버 설정**
2. WebSocket을 활성화합니다. 비밀번호를 설정한 경우 메모해 두세요
3. EsperantAI에서: **연결** 패널
4. 스트리밍 소프트웨어: **OBS Studio**
5. WebSocket URL: `ws://127.0.0.1:4455` (기본값)
6. 비밀번호: OBS에서 설정한 것
7. **Connect** 클릭

### Streamlabs Desktop

1. Streamlabs에서: **Settings → Remote Control**
2. Remote Control 활성화
3. API Token을 메모
4. EsperantAI에서: 스트리밍 소프트웨어: **Streamlabs Desktop**
5. API Token: 붙여넣기
6. 포트: `59650` (기본값)
7. **Connect** 클릭

### vMix

1. vMix에서: **Settings → Web Controller**
2. Web Controller 활성화. 기본 포트: 8088
3. EsperantAI에서: 스트리밍 소프트웨어: **vMix**
4. 호스트: `127.0.0.1`
5. 포트: `8088`
6. **Connect** 클릭

### PRISM Live Studio

1. PRISM Live Studio v4.0.5+는 obs-websocket 플러그인을 수동으로 설치해야 합니다
2. [OBS 포럼](https://obsproject.com/forum/resources/obs-websocket-remote-control-of-obs-studio-made-easy.466/)에서 `obs-websocket` 다운로드
3. PRISM의 플러그인 폴더에 복사
4. PRISM 재시작
5. **도구 → WebSocket 서버 설정** 에서 WebSocket 활성화
6. EsperantAI에서: 스트리밍 소프트웨어: **PRISM Live Studio** (OBS와 동일하게 작동)

### XSplit Broadcaster (베타)

1. XSplit에서 "Remote xjs" 확장 프로그램 설치 (Settings → Extensions)
2. 환경설정에서 Remote 활성화
3. EsperantAI에서: 스트리밍 소프트웨어: **XSplit**
4. Remote xjs 프록시 URL: `ws://127.0.0.1:5555/xjs` (기본값)
5. **Connect** 클릭

> XSplit은 **베타**입니다. 고급 기능이 제한될 수 있습니다.

## 제스처 및 씬 설정

연결하면 스트리밍 소프트웨어의 실제 씬이 **트리거** 패널의 드롭다운에 자동으로 나타납니다.

### 기본 매핑

1. 각 제스처(예: "왼쪽 보기")에 대해 드롭다운에서 씬을 선택
2. 해당 제스처를 취하고 약 150ms 동안 안정적으로 유지하면, EsperantAI가 스트리밍 소프트웨어의 씬을 전환합니다
3. 전환은 자동이며 거의 즉각적입니다

### 멀티 액션 (Pro+)

Pro 또는 Pro+ 라이선스가 있으면 하나의 제스처로 **여러 동작**을 동시에 트리거할 수 있습니다:
- 씬 전환 + 소리 재생 + 오버레이 표시 + 채팅 메시지 전송

### 카테고리 활성화/비활성화

각 카테고리에는 개별 "활성화" 체크박스가 있습니다:

- 🧠 **머리 회전** (유니버설 — 기본 활성화)
- 📏 **얼굴 거리** (가까이/멀리 이동)
- 👁️ **시선** (눈만 이동)
- 😀 **감정** (미소, 놀람, 분노, 중립)
- 👁️‍🗨️ **더블 블링크**
- ✋ **손 제스처** (컬처럴 — 기본 비활성화)

CPU를 절약하려면 필요하지 않은 카테고리를 비활성화하세요.

## 제스처 카테고리

### 🌐 유니버설 (모든 문화에서 동일한 의미)

| 제스처 | 축 | 활성화 방법 |
|---|---|---|
| 센터 | — | 정면을 보고 얼굴을 안정적으로 유지 |
| 왼쪽 보기 | 음의 yaw | 머리를 왼쪽으로 돌리기 |
| 오른쪽 보기 | 양의 yaw | 머리를 오른쪽으로 돌리기 |
| 위 보기 | 음의 pitch | 얼굴을 올리기 |
| 아래 보기 | 양의 pitch | 얼굴을 내리기 |
| 왼쪽 기울이기 | 음의 roll | 머리를 왼쪽 어깨 쪽으로 기울이기 |
| 오른쪽 기울이기 | 양의 roll | 머리를 오른쪽 어깨 쪽으로 기울이기 |
| 가까이 다가가기 | 거리 | 얼굴을 카메라에 가까이 |
| 멀리 물러나기 | 거리 | 얼굴을 카메라에서 멀리 |
| 시선 | 시선 | 머리는 고정하고 눈만 이동 |
| 미소 | 감정=happy | 뚜렷하게 미소 짓기 |
| 놀람 | 감정=surprise | 놀란 표정 짓기 |
| 분노 | 감정=angry | 화난 표정 짓기 |
| 중립 | 감정=neutral | 편안한 표정 |
| 더블 블링크 | 블링크 | 양 눈을 빠르게 두 번 감기 (700ms 이내) |

### ⚠️ 컬처럴 (국가에 따라 의미가 다를 수 있음)

| 제스처 | 서양에서의 의미 | 다른 문화에서의 주의사항 |
|---|---|---|
| 👍 엄지 척 | 찬동, 좋음 | 중동/서아시아: 모욕적으로 받아들일 수 있음 |
| ✌️ 브이 피스 | 평화/승리 | 영국/아일랜드/호주 (손바닥 안쪽): 모욕. ※한국에서는 사진 찍을 때 매우 흔하고 긍정적인 제스처로 사용됨 |
| 🤘 락 사인 | 록/메탈 | 이탈리아 (손바닥 아래): "cornuto" (모욕) |
| 👌 OK | OK/완벽 | 브라질/터키/독일: 모욕적으로 받아들일 수 있음. ※한국에서는 매우 불쾌한 제스처이므로 사용에 주의 필요 |
| ✊ 주먹 쥐기 | 정치적 맥락에 따라 다름 | — |
| 🖐️ 손바닥 펼치기 | "멈춤" 또는 인사 | 그리스 (상대에게 mountza): 강한 모욕 |
| ☝️ 손가락 가리키기 | 지시 | 아시아: 손가락으로 가리키는 것은 무례 |

EsperantAI는 UI에서 각 제스처에 해당하는 배지를 표시합니다. 글로벌 시청자를 고려하여 어떤 제스처를 사용할지 선택하세요.

### 🙏 합장 (합장)

특별한 제스처: 가슴 앞에서 양 손바닥을 맞대어 모읍니다 (기도나 인사처럼). 합장은 한국 문화에서 인사, 감사 표현, 불교 기도 등에 널리 사용되는 친숙한 제스처입니다. 일상생활에서도 감사나 죄송함을 표현할 때 자연스럽게 사용됩니다. 6개의 랜드마크 체크를 통해 높은 신뢰도로 감지됩니다.

## 스트리밍 플랫폼 연결

EsperantAI가 이벤트(도네이션, 구독, 레이드)를 수신하려면, 방송 중인 플랫폼을 연결하세요.

### Twitch

1. https://dev.twitch.tv/console 에서 Client ID 생성
2. 리디렉트 URI 등록: `https://edugame.digital/oauth-callback.html` (또는 로컬 URL)
3. EsperantAI에서: **플랫폼 이벤트** 패널 → **Twitch EventSub**
4. Client ID 붙여넣기
5. **Connect** 클릭
6. Twitch 인증 창이 열립니다. 권한을 허용하세요
7. 창이 닫히고 "Twitch Connected"가 표시됩니다

### YouTube Live

1. https://console.cloud.google.com 에서 인증 정보 생성
2. YouTube Data API v3 활성화
3. OAuth Client ID 생성 (유형: 웹 애플리케이션)
4. Twitch와 동일한 리디렉트 URI 등록
5. EsperantAI에서: **플랫폼 이벤트** 패널 → **YouTube Live**
6. Client ID를 붙여넣고 **Connect** 클릭

### Kick

1. https://kick.com/settings/developer 에서 앱 생성
2. 리디렉트 URI 등록
3. EsperantAI에서: **플랫폼 이벤트** 패널 → **Kick**
4. Client ID를 붙여넣고 **Connect** 클릭
5. Kick은 OAuth 2.1 with PKCE를 사용합니다 (더 안전)

### StreamElements (멀티플랫폼 브릿지)

이미 StreamElements 계정이 있다면 하나의 토큰으로 Twitch + YouTube + Facebook을 통합할 수 있습니다:

1. https://streamelements.com/dashboard/account/channels 에 접속
2. JWT Token 복사
3. EsperantAI에서: **플랫폼 이벤트** 패널 → **StreamElements**
4. JWT를 붙여넣고 **Connect** 클릭

## 이벤트 + 제스처 콤보 (고급)

EsperantAI의 핵심 기능: **플랫폼 이벤트**와 **사용자의 제스처**를 확인으로 결합합니다.

### 예시: 도네이션에 엄지 척으로 감사 표현

1. **이벤트 트리거** 패널 → "💰 도네이션" 행
2. ✅ 활성화
3. 씬: `Thank_You_Scene`
4. 필수 제스처: `👍 엄지 척`

**라이브 흐름**:
- 도네이션이 들어옴 → EsperantAI에 "제스처 대기 중..." 표시
- 5초 이내에 👍을 취함
- 취하면 → `Thank_You_Scene`으로 전환 + 설정된 다른 동작 실행
- 취하지 않으면 → 자동으로 취소

### 필수 제스처 없음 (자동 트리거)

"필수 제스처"를 `— 없음 —`으로 그대로 두면 이벤트가 즉시 동작을 트리거합니다.

유용한 경우:
- 레이드가 들어올 때 자동으로 축하 씬으로 전환
- 누군가 구독할 때 자동으로 오버레이 표시

## 감도 및 데드존

### 감도

임계값은 제스처가 트리거되려면 얼마나 크게 해야 하는지를 제어합니다:

- **Yaw**: 머리를 옆으로 얼마나 돌리는지 (기본값: 0.15 rad ≈ 8.6°)
- **Pitch 위/아래**: 수직 기울기
- **Roll**: 측면 기울기

값을 올리면 더 과장된 제스처가 필요합니다. 내리면 더 민감해집니다.

### 데드존 (피로도 방지)

거의 중앙을 보고 있을 때 (yaw < 0.05, pitch < 0.05, roll < 0.08) **아무것도 트리거되지 않습니다**. 이를 통해 미세한 움직임이 트리거를 활성화하는 것을 방지하고 자연스럽게 움직일 수 있습니다.

### 안정 프레임

`안정 프레임` = 트리거되기 전에 제스처를 연속해서 몇 프레임 동안 유지해야 하는지. 기본값: 5프레임 (약 150ms / 30fps).

트리거가 너무 쉽게 발동하면 값을 올리고, 더 빠른 반응이 필요하면 내리세요.

### 쿨다운

`쿨다운 (ms)` = 씬 전환 사이의 최소 시간. 기본값: 500ms.

빠르게 흔들렸을 때 전환이 "떨리는" 것을 방지합니다.

## 문제 해결

### 라이선스 키를 붙여넣었는데도 "활성화 필요"가 계속 표시됨

- 키를 완전히 복사했는지 확인 (하이픈으로 구분된 4자리 5그룹)
- 인터넷 연결 확인 (첫 활성화에는 온라인 검증이 필요)
- 이미 3대에서 활성화한 경우, 먼저 한 대를 비활성화
- 계속되면 soporte@edugame.digital으로 연락

### 얼굴이 보이는데도 "얼굴 검색 중..."이 계속됨

- 조명 개선: 얼굴이 밝게 조명되어야 합니다
- 카메라에 가까이 이동 (40-80cm가 최적)
- GPU를 사용하는 다른 탭 닫기 (Chrome은 탭이 너무 많으면 GPU를 제한할 수 있음)
- Chrome의 메모리 절약 기능이 활성화된 경우 이 탭에서 비활성화

### 드롭다운에 씬이 나타나지 않음

- 스트리밍 소프트웨어에 연결되어 있는지 확인 (초록색 배지 "Connected")
- `R` 키를 눌러 씬 목록 새로고침
- 여전히 비어있으면 연결을 끊고 다시 연결

### 제스처를 하지 않았는데 씬 전환이 트리거됨

- **감도** 패널에서 yaw / pitch / roll 임계값 올리기
- `안정 프레임`을 5에서 8-10으로 올리기
- 데드존이 설정되어 있는지 확인 (yaw 0.05, pitch 0.05, roll 0.08)
- 화면에 다른 사람이 없는지 확인 (다중 얼굴로 불안정해질 수 있음)

### 감지 지연

- 무거운 앱 (게임, 영상 편집) 닫기
- 내장 그래픽이 아닌 전용 GPU를 사용하고 있는지 확인 (있는 경우)
- 카메라 해상도가 4K인 경우 1080p로 낮추기 (감지에는 1080p가 최적)

### EsperantAI에 "씬 변경됨"이라고 표시되는데 OBS가 반응하지 않음

- 드롭다운의 씬 이름이 OBS의 것과 정확히 일치하는지 확인 (대소문자 구분)
- 씬이 다른 씬 컬렉션에 있지 않은지 확인
- **트리거 기록** 패널 확인 — ✗ 빨강이 표시되어 있으면 구체적인 오류가 있습니다

### "OBS에 연결할 수 없음 — 수동으로 연결하세요" 오류

- OBS가 열려 있는지 확인
- OBS에서 WebSocket이 활성화되어 있는지 확인
- OBS에서 비밀번호를 설정한 경우, 정확히 일치해야 합니다
- 일부 백신 소프트웨어가 포트 4455를 차단 — 예외 추가
